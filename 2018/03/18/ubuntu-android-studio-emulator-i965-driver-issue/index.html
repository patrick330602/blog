网上搜索了一番，发现这个问题在不同版本Ubuntu 16.10+都有出现，原因很简单：谷歌为了防止在不同系统上为了防止出现兼容性问题，用了自己的libstdc++包，但这些写死的包却不一定支持用户所对应的系统，比如我就是用的自定义的Surface Book内核。虽然这个问题是已知的，但谷歌似乎还是没能给出一个完美的解决方案。其实，解决问题很简单：使用系统的libstdc+就行了。
